# 协同调控：UI/交互与状态机规格（参数面板 + 点选编辑 + 参数提取）

日期：2026-01-27

> 本规格以“右侧协同调控参数面板”为中心：导入设计工作面（4点一组）、生成评价点、按工作面点选编辑布局参数/生产参数、参数提取两阶段（煤厚 vs 采高拆分）、以及撤销/重做接入。

## 1. 右侧面板信息架构

### 1.1 放置位置
- 右侧 Accordion 中的 Section：协同调控。

### 1.2 面板目标（用户视角）
- 我能导入“设计工作面坐标”（任意四点一组），系统自动排序成一个平行四边形/矩形，并在主图显示。
- 我能生成评价点（复用 Summary 的步长与生成逻辑），并在主图显示点位。
- 我能点击主图中的工作面轮廓，系统命中哪一个就编辑哪一个；若导入面与规划面同时命中，弹出小选择“编辑导入面 / 编辑规划面”。
- 我能按工作面分别保存：布局层参数（工作面宽度/区段煤柱宽度/设计推进长度）与生产层参数（采高/推进长度/推进速度）。
- 我能在生产层通过“确认”一键保存本工作面的生产参数，并用于后续快速复算。
- 我能在生产层看到该工作面区域内的平均煤厚，以及对应的设计推进长度。
- 我明确“煤厚”和“采高”是两条口径：煤厚来自所选煤层 `selectedCoal` 的钻孔厚度场插值；采高来自生产层的“设计采高”。生产层确认/保存时触发一次“按采高重提取”（不影响煤厚）。
- 我可以用撤销（Ctrl+Z 或界面撤销）回退一次工作面编辑/参数编辑。

## 2. 数据模型与主键

### 2.1 两类工作面
- 导入工作面（ImportWorkface）：来自 CSV/文本导入，每 4 个点为 1 个工作面，编号为 `No.k`（k 从 1 开始）。
- 规划工作面（PlannedWorkface）：来自规划模块（workface loops），编号为 `faceIndex`。

### 2.2 参数存储（两套表分开存）
- 导入面参数表：按 `No.k` 作为 key。
- 规划面参数表：按 `faceIndex` 作为 key。

建议持久化结构（示意）：

```json
{
  "import": {
    "No.1": { "layout": {"workfaceWidthM": 240, "sectionPillarWidthM": 20, "advanceLengthM": 1000}, "production": {"miningHeightM": 4, "advanceLengthM": 200, "advanceSpeedPerShift": 6} }
  },
  "planned": {
    "1": { "layout": {"workfaceWidthM": 240, "sectionPillarWidthM": 20, "advanceLengthM": 1000}, "production": {"miningHeightM": 4, "advanceLengthM": 200, "advanceSpeedPerShift": 6} }
  }
}
```

> 注：这里仅规定“分表+主键口径”。具体落地的 localStorage key 与默认值沿用项目现有协同调控存储策略，但需要把导入/规划分开。

## 3. 4点一组：自动排序与合法性

### 3.1 输入规则
- 每 4 点视为一个工作面角点集合，允许任意顺序。

### 3.2 排序规则（输出一个四边形）
- 计算 4 点质心 $C$。
- 按 $\theta=\operatorname{atan2}(y-C_y,\ x-C_x)$ 对 4 点做角度排序，得到顺时针/逆时针顶点序列。
- 验证：四边形边不自交、面积不接近 0。
  - 若出现自交：尝试交换对角点（典型修复：交换序列中 2、3）再验证。
  - 若仍非法（例如 1 点落在三角形内部导致无法形成凸四边形）：该工作面标记为 invalid，并在 UI 给出提示“4点无法构成有效工作面轮廓”。

### 3.3 走向/面宽定义（口径已确认）
- “工作面宽度（m）” = 四边形的短边长度（实时几何计算）。
- “推进长度（m）” = 四边形的长边长度（实时几何计算）。
- 计算方法：取排序后四边形相邻边的长度，计算两组对边的平均长度：
  - $L_0=\frac{1}{2}(|P_0P_1|+|P_2P_3|)$
  - $L_1=\frac{1}{2}(|P_1P_2|+|P_3P_0|)$
  - 工作面宽度 $B=\min(L_0,L_1)$；推进长度 $L=\max(L_0,L_1)$。

## 4. 点选命中与编辑目标

### 4.1 点选命中规则
- 点击主图时，系统分别对“导入工作面集合”和“规划工作面集合”做命中检测（点在多边形内，或离边线距离 < 阈值）。
- 命中集合：
  - 仅命中导入面：直接进入“编辑导入面（No.k）”。
  - 仅命中规划面：直接进入“编辑规划面（faceIndex）”。
  - 两者同时命中：弹出小选择（就地 Popover/Modal）：
    - “编辑导入面”
    - “编辑规划面”

### 4.2 进入编辑后的联动
- 右侧协同调控面板自动滚动到“协同调控参数（布局层）”。
- 布局层输入框回填为“该工作面自身”的参数：
  - 若参数表中存在：读对应 entry。
  - 若不存在：用全局默认值初始化，但写入时仍写到该工作面的 entry。

（新增，2026-01-27）生产层辅助信息：
- 选中工作面后，生产层展示两项只读信息：
  - 平均煤厚（m）：基于当前煤厚插值场 `CoalThk` 在工作面区域内采样求均值（采样步长自适应，最大采样数上限）。
  - 设计推进长度（m）：使用布局层“设计推进长度（m）”的当前值（若为空则用几何尺寸推导值）。
- 选中工作面后，生产层会把上述信息同步回填到对应输入框（仅当该工作面生产层参数仍为默认/空值时才回填，避免覆盖用户手动修改）：
  - 设计采高（m）= 平均煤厚（m）
  - 工作面推进长度（m）= 设计推进长度（m）
- 生产层提供“ODI计算”按钮：按当前调控参数执行“全参提取 + ODI 计算”，用于调参后的快速复算。

## 5. 工作面轮廓更新（仅更新自身，且以平移为主）

> 口径已确认：“更新该工作面自身轮廓，进行平移”。因此修改布局参数时不得重算全局工作面集合。

### 5.1 工作面宽度（短边）更新
- 宽度 $B$ 改变时：优先沿“短边法向”调整，保持工作面整体方向不抖动。
- 锁定策略（已确认）：
  - 若仅存在一侧相邻工作面：锁定“更靠近相邻工作面”的那一侧边，另一侧按目标宽度伸缩。
  - 若两侧都有相邻工作面：锁定“自身中心线”（即两侧边对称移动，保持中心不变）。

### 5.2 区段煤柱宽度（ws）更新
- `ws` 不再是单纯手填后直接平移的量，而是“选定基准侧 + 邻近工作面”驱动的自动计算值。
- 用户需选择：`ws 基准侧`（沿 x 布置时：上/下；沿 y 布置时：左/右）。
- 邻近工作面集合：同时考虑导入面与规划面（若同时命中多个候选，弹窗让用户选定“基准工作面”）。
- 重叠判定（已确认）：推进方向投影“任意重叠”即视为候选邻面。
- ws 的数值口径：当前工作面与所选“基准工作面”在宽度方向上的净间距（gap）。

### 5.3 推进长度（布局层设计 L）更新
- 布局层新增字段：`advanceLengthM`（设计推进长度，和生产层的动态推进长度分开）。
- 方向口径：推进长度方向与工作面宽度方向垂直（即沿长边方向）。
- 几何约束：调整推进长度时，工作面宽度保持不变；以“推进长度基准端”固定一端，仅移动另一端的端边进行伸缩。
- 右侧增加：`推进长度基准端` 选项（已确认自动提供）：
  - 若沿 x 布置：左端 / 右端
  - 若沿 y 布置：上端 / 下端
- 更新规则：基准端固定，另一端沿推进方向伸缩，使得长边达到目标推进长度。

### 5.4 工作面方向判定与锁定
- 默认：系统自动判定工作面“沿 x / 沿 y 布置”。
- 选中工作面后：系统会按自动判别结果直接“锁定方向”（作为后续 ws/B/L 口径的基准）。
- 若自动判别不正确：UI 提供“翻转”操作（x↔y），由用户纠正；纠正后保持锁定，不再随几何微调跳变。

## 8. 选中工作面后的自动更新逻辑（ws/B/L）

### 8.1 方向判定（axis-aligned）
- 默认自动：
  - 若 $|\max X-\min X| \ge |\max Y-\min Y|$，判定“沿 x 布置”（推进方向为 x、宽度方向为 y）。
  - 否则判定“沿 y 布置”（推进方向为 y、宽度方向为 x）。
- 若开启“锁定方向”：方向取锁定值，不再随几何变化自动切换。

### 8.2 邻近工作面候选（两者都算）
- 候选集合：导入面 + 规划面（统一在世界坐标下比较）。
- 推进方向重叠判定（已确认）：推进方向投影区间“任意重叠”即算候选。
- 基准侧过滤：根据用户选择的 `ws 基准侧`（上/下/左/右）只保留该侧的候选工作面。

### 8.3 多候选弹窗（基准工作面选择）
- 若过滤后候选数量为 0：ws 不自动回填；UI 给出提示“未找到该侧相邻工作面”。
- 若候选数量为 1：直接作为基准工作面。
- 若候选数量 ≥ 2：弹窗列出候选（展示：导入/规划、编号、gap 值），用户选定一个作为基准工作面。

### 8.4 ws 计算（gap）
- ws = 当前工作面与“基准工作面”在宽度方向上的净间距（gap）。
  - 沿 x 布置（宽度方向 y）：
    - 基准侧=上：ws = neighbor.minY - cur.maxY
    - 基准侧=下：ws = cur.minY - neighbor.maxY
  - 沿 y 布置（宽度方向 x）：
    - 基准侧=右：ws = neighbor.minX - cur.maxX
    - 基准侧=左：ws = cur.minX - neighbor.maxX

### 8.5 宽度 B 更新（锁边/锁中心）
- 若仅存在一侧相邻工作面（以“ws 基准侧”与基准工作面为准）：锁定更靠近基准工作面的那一侧边，另一侧按目标宽度伸缩。
- 若两侧都有相邻工作面：锁定自身中心线（两侧对称移动）。

### 8.6 推进长度 L 更新（基准端）
- UI 提供“推进长度基准端”：
  - 沿 x：左端（锁 minX）/右端（锁 maxX）
  - 沿 y：下端（锁 minY）/上端（锁 maxY）
- 更新时基准端固定，另一端伸缩到目标推进长度。

## 6. 参数提取：煤厚与采高彻底拆开（两阶段）

### 6.1 口径总览
- 煤厚（coalThickness）：来自当前所选煤层 `selectedCoal`，并且在评价点上通过“煤厚地质钻孔厚度场插值”得到每点煤厚。
- 采高（miningHeight）：来自生产层字段“设计采高”。
- 两者不得互相覆盖：生产层确认/保存触发“按采高重提取”，不得改变煤厚。

### 6.2 布局层提取（评价点级）
- 输入：评价点集合、`selectedCoal`、钻孔分层数据。
- 输出：每个评价点附带 `coalThickness`（来自厚度场插值）以及其余“全参插值/地质插值”参数（按既有提取口径）。

### 6.3 生产层确认/保存触发的二次提取
- 触发时机：生产层“确认/保存”（右侧“推进速度”行的“确认”按钮，或底部“保存”）。
- 输入：上一阶段的评价点集（含煤厚） + 生产层采高。
- 输出：更新与采高相关的派生参数/评分/ODI 等（具体以既有实现为准），并明确：不更新煤厚字段。

## 7. 撤销/重做接入

### 7.1 接入原则
- 修改“工作面参数表”（导入/规划任一）或“工作面轮廓几何”时，需要进入撤销栈。
- 目标：用户按 Ctrl+Z 能回退一次点选编辑带来的状态变化。

### 7.2 最小可验收标准
- 调整任意一个工作面的宽度/ws/推进长度（布局层）后，按 Ctrl+Z：
  - 该工作面的轮廓恢复到上一步
  - 该工作面的参数恢复到上一步
  - 其他工作面不受影响
