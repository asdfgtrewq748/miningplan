# 功能联动机制原理图（代码检索版）

> 目标：用“数据对象 + 调用关系”把以下板块串成一条可追溯链路：
> 多场景覆岩扰动综合评价 → 地质参数分析 → 采区规划 → 协同调控 → 采掘接续 → 工程经济分析。
>
> 主控文件：frontend/src/App.jsx（集中式状态机 + 各模块的计算与跳转）。

## 2) 前端关键“中间结果对象”（便于对照日志/快照）

- 参数提取
	- paramExtractionGeoResult：地质插值提取结果（Ti/Ei/Hi/Di/Mi...）
	- paramExtractionFullResult：全参+开采参数提取结果（含 lpi/lci/delta 等）
	- paramExtractionResult：当前显示/导出的“最近一次提取”（会被 geo/full 覆盖）
- 扰动/ODI
	- odiResult：ODI_raw 与归一化（odiNorm）等
	- odiFieldPack：ODI 插值场（0~1；可视化可再次归一化）
- 协同调控
	- coScalePack：ODI* 统一标尺（qLo/qHi 与端点值 qLowValue/qHighValue）
	- coOdiUnionResult：unionPoints 的 ODI 计算点集（区分 geoPoints/fullPoints）
	- coOdiAnalysisResult：ODI* 网格采样后的工作面统计 + 推进曲线
- 采区规划
	- planningEfficiencyResult / planningResourceResult / planningWeightedResult
	- plannedWorkfaceLoopsWorld / plannedWorkfaceUnionLoopsWorld
- 采掘接续
	- successionSource：接续输入来源选择（规划 vs 协同调控）
	- successionPanels：由 loops 提取的面宽/推进长度/中心点
	- successionStage1Plan：阶段1排程（tasks + monthly）
	- successionStage2Risk：阶段2风险序列（字段可来自 ODI 或 ODI*）
	- successionStage3Result：阶段3推荐/对比结果
- 工程经济
	- economicsResult：现金流、NPV、回收期、单位成本等

## 3) 后端接口（Python FastAPI 主链路）

- 规划计算（frontend/src/api.js → backend_python/routers/planning.py）
	- POST /api/planning/smart-efficiency/compute
	- POST /api/planning/smart-resource/compute
	- POST /api/planning/smart-weighted/compute（可多目标 + 可选引入扰动字段）
	- POST /api/planning/smart-resource/tonnage
- 接续优化（可选，与前端阶段1“本地排程”并存）
	- POST /api/succession/quick-optimize 等（backend_python/routers/succession.py）

## 4) 备注（代码结构现实）

- 前端 App.jsx 既是 UI 也是“总调度/总状态机”，多模块的结果对象在这里互相引用。
- 仓库里还有一套 Node/Express 的 backend 目录（backend/、server/），但当前前端默认 API_BASE 指向 3001 的 Python FastAPI（backend_python）。
