# 采区规划设计项目论文证据矩阵
更新日期：2026-03-25

## 1. 新版系统识别证据
- 结论：当前论文应以新版 `GeoMind NEXUS` 智能采掘设计系统为主线。
  证据：线上首页源码 `05_支撑材料/线上新版系统首页.html` 与本地 `frontend/dist/index.html` 标题一致，均为“GeoMind NEXUS 智能采掘设计系统”。
- 结论：新版系统主入口是 `frontend + backend_python`。
  证据：`frontend/src/App.jsx` 组织了主界面状态、三场景分析、协同调控、智能规划、接续和经济分析；`backend_python/main.py` 注册了 upload、boreholes、design、score、boundary、geology、succession、gnn、planning、export 路由。

## 2. 新旧系统边界证据
- 结论：`backend/` 与 `server/` 为旧版 Node 原型系统。
  证据：`backend/index.js` 与 `server/index.js` 仅注册 boundary、boreholes、score、design 等基础路由，不具备新版平台级 planning 与接续主线。
- 结论：新版系统在功能范围上明显超出旧版基础设计系统。
  证据：`frontend/src/App.jsx` 中 `mainViewMode` 覆盖 `odi`、`geology`、`planning`、`cocontrol`、`succession`、`economics`；`VALID_SCENARIO_TABS` 包括 `surface`、`aquifer`、`upward`。

## 3. 新版平台功能证据
- 结论：新版系统包含三场景分析、ODI、协同调控、智能规划、接续与经济分析等平台级能力。
  证据：
  - 三场景与视图主线：`frontend/src/App.jsx`
  - 多目标规划面板：`frontend/src/components/MultiObjectivePlanPanel.jsx`
  - 接续阶段视图：`frontend/src/components/SuccessionStage1View.jsx`
  - 工程经济分析：`frontend/src/components/EconomicsView.jsx`
  - 四模型规划后端：`backend_python/routers/planning.py`
- 结论：智能规划是“四模式”而非单一布局算法。
  证据：`docs/智能规划_四模型算法说明.md`；`backend_python/routers/planning.py` 暴露 `smart-efficiency`、`smart-resource`、`smart-weighted`、`smart-resource/tonnage` 等接口。
- 结论：接续模块不仅存在基础接口，还具有训练、优化、快速优化等扩展能力。
  证据：`backend_python/routers/succession.py`。

## 4. 样例运行证据
- 结论：系统可完成边界导入、钻孔加载、地质建模和采区设计的样例级贯通运行。
  证据：`05_支撑材料/接口结果/边界数据.json`、`钻孔数据.json`、`地质建模结果.json`、`采区设计结果.json`。
- 结论：样例设计结果生成了 3 个工作面和 11 条巷道。
  证据：`05_支撑材料/接口结果/采区设计结果.json`。
- 结论：系统具备 GNN 地质预测的接口与网格输出能力。
  证据：`05_支撑材料/接口结果/GNN训练结果.json`、`GNN网格结果.json`。

## 5. 图件证据
- 结论：系统具备从空间输入到设计输出的图件表达能力。
  证据：`01_可视化图汇总/系统生成图/图1_采区边界与钻孔分布图.png`、`图2_煤层厚度插值热力图.png`、`图3_采区工作面与巷道布局图.png`、`图4_GNN煤层厚度预测网格图.png`。

  证据：`02_总流程图/采掘规划系统总流程图.svg`、`03_示意图初稿/*.svg`、`00_过程文档/新旧规划系统区分说明.md`。

## 6. 已验证结论与边界说明
- 已验证：
  - 新版系统线上身份与本地 `frontend/dist` 一致。
  - 新版系统在代码层面已形成三场景、ODI、协同调控、智能规划、接续、经济分析的完整平台结构。
  - 样例数据可跑通边界导入、钻孔处理、地质建模、采区设计和 GNN 输出链路。
  - 论文支撑图件、接口结果和过程文档已完成统一归档。
- 仅作工程展示，不作强结论：
  - GNN 指标仅基于小样本示例，不能外推为稳定泛化性能。
  - 评分与部分综合评价接口仍存在原型性质，不宜写成成熟工业评价模块。
  - 接续与经济分析虽然在新版系统中已形成明显模块，但当前论文仍以结构和实现链路说明为主，尚未完成真实矿井参数标定。
