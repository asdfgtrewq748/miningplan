# 采区规划设计项目小论文任务规格说明
更新日期：2026-03-25

## 1. 任务目标
围绕本项目已实现的采区规划设计系统，形成一套可直接用于论文初稿整理的中文材料，覆盖以下 5 类交付物：

1. 项目产出的可视化图件汇总，用于证明系统具备从数据导入、建模、规划、评价到导出的完整可行性。
2. 采掘规划系统总流程图，用统一流程串联各子模块。
3. 论文示意图初稿，采用 `SVG` 矢量格式，并配套图名与图意说明文档。
4. 小论文正文初稿，同时输出 `Markdown` 版本与 `DOCX` 版本。
5. 任务执行过程的规格、进度、验收标准与证据矩阵。

## 2. 工作范围

### 2.1 纳入论文叙述的系统范围
- 数据导入与标准化：采区边界、钻孔坐标、钻孔属性、示例样本组织。
- 地质建模：规则网格建模、厚度空间场生成、GNN 地质预测。
- 采区设计：工作面布局、巷道生成、几何约束校核。
- 智能规划：多模型规划逻辑、资源与效率导向的方案生成。
- 指标评价：布局指标、扰动风险、回采率、接续与工程经济分析。
- 后处理与导出：CAD 导出、图件输出、结果归档。

### 2.2 本轮交付边界
- 以仓库现有代码、文档、样例数据、快照与补充运行结果为主要证据来源。
- 对于已能稳定运行并产出结果的模块，按“功能链路贯通 + 图件验证”组织表述。
- 对于仍处于原型或占位状态的模块，仅表述为“工程实现能力展示”，不表述为严格定量验证结论。
- 论文按“中文小论文初稿”标准组织，不在本轮承诺终稿级投稿润色。
- 参考文献不允许虚构；当前仓库缺少稳定外部文献清单，因此正文保留“待补充”处理。

## 3. 交付目录约定
- `00_过程文档`：规格说明、进度追踪、验收标准、证据矩阵。
- `01_可视化图汇总`：系统生成图、仓库已有图、图件索引。
- `02_总流程图`：系统总流程图 `SVG` 与文字说明。
- `03_示意图初稿`：各类示意图 `SVG` 与图意说明。
- `04_论文稿件`：论文 `Markdown`、论文 `DOCX`。
- `05_支撑材料`：接口结果、图件脚本、文档生成脚本、运行日志。

## 4. 论文组织原则
- 论文主题聚焦“采区规划设计系统”的完整技术链路，而非单一算法。
- 主体结构遵循中文科技论文常见写法：题名、摘要、关键词、引言、系统与方法、结果与验证、讨论、结论、参考文献。
- 所有关键结论必须能回溯到代码、图件、接口结果或已有项目文档。
- 对未完成严格生产验证的内容，只能表述为“系统实现能力”或“样例级可行性展示”。
- 当前论文以 `backend_python` 为主实现主线，旧版 `backend` 与 `server` 仅作为项目演进背景。

## 5. 本轮执行策略

### 5.1 证据优先
1. 梳理系统主链路与关键代码入口。
2. 用样例数据复现“边界导入 - 钻孔加载 - 地质建模 - 采区设计 - GNN 预测”链路。
3. 从运行结果和仓库历史图件中筛选论文可用图件。
4. 在证据清单稳定后再撰写论文正文。

### 5.2 图件优先级
1. 可复现的系统生成图。
2. 仓库中已有、且与系统能力直接对应的图件。
3. 基于代码结构绘制的系统流程图与示意图。

### 5.3 文档生成策略
- `Markdown` 作为主编写源文件。
- `DOCX` 作为交付版式文件，由本地脚本自动生成。
- 作者、单位、基金项目、中图分类号等无法从仓库自动推断的信息以中文占位方式保留。

## 6. 风险与依赖
- 系统中存在多后端并存的历史结构，论文叙述必须统一主线，避免混写。
- 当前 GNN 样例数据规模较小，训练指标只能作为样例展示，不能上升为普适性能结论。
- 评分模块中存在原型接口，论文验证重点应放在“规划链路可跑通”和“图件可产出”。
- 如需严格投稿模板或作者信息定稿，仍需用户后续补充。
