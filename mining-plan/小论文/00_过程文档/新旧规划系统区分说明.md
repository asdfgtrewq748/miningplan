# 新旧规划系统区分说明
更新日期：2026-03-25

## 1. 目的
上一版论文初稿对项目系统边界识别不够准确，存在以旧版规划链路替代新版主系统叙事的问题。为避免后续论文继续混写，本说明将项目中的“新版系统”和“旧版系统”明确区分，并给出本轮论文重写采用的唯一主线。

## 2. 新版系统认定

### 2.1 线上系统识别
- 用户提供的线上地址为 `http://39.97.168.66/`。
- 已抓取线上首页源码，页面标题为“GeoMind NEXUS 智能采掘设计系统”。
- 本地 `frontend/dist/index.html` 标题与线上标题一致，说明线上系统对应当前仓库中的新版前端发布结果。

### 2.2 新版系统代码主线
- 前端主线：`frontend/src/App.jsx`
- 后端主线：`backend_python/main.py`
- 规划后端：`backend_python/routers/planning.py`
- 接续后端：`backend_python/routers/succession.py`
- 经济分析前端：`frontend/src/components/EconomicsView.jsx`
- 接续阶段前端：`frontend/src/components/SuccessionStage1View.jsx`
- 多目标规划前端：`frontend/src/components/MultiObjectivePlanPanel.jsx`

### 2.3 新版系统的真实功能边界
新版系统不是单纯的“边界导入 - 地质建模 - 工作面布局 - 导出”链路，而是面向 `GeoMind NEXUS` 平台化的一体化系统，至少包含以下模块：

1. 多源数据导入与项目快照管理。
2. 地质建模与地质预览。
3. 三场景分析主线：`surface`、`aquifer`、`upward`。
4. ODI 计算、插值、排序和分级响应。
5. 协同调控分析，包括 ODI 分析面板、圈定范围对比、批量 ODI 计算、ODI* 统一标尺。
6. 智能采区规划四模式：工程效率、资源回收、覆岩扰动、权重综合。
7. 采掘接续多阶段分析。
8. 工程经济分析，包括按月现金流、净现值、回收期、单位成本、高风险月统计。
9. CAD/图件导出与项目归档。

## 3. 旧版系统认定

### 3.1 旧版目录
- `backend/`
- `server/`

### 3.2 旧版系统特征
- 均为 Node/Express 结构。
- 主要接口集中在 `boundary`、`boreholes`、`score`、`design`，部分版本包含 `upload`、`geology`。
- 不包含新版系统中的 `planning` 四模型、平台级协同调控主链、接续阶段 UI 主线和新版工程经济分析主界面。
- 其价值主要在于说明项目的早期原型与演进历史，不应作为当前论文主系统。

## 4. 本轮论文采用的写作原则
- 论文唯一主系统：`GeoMind NEXUS` 新版智能采掘设计系统。
- 旧版 Node 目录只在“系统演进与版本边界”章节中说明，不再承担正文主叙事。
- 正文模块必须覆盖新版系统的真实功能链，而不仅是基础设计链。
- 样例运行结果仍可作为可行性验证证据，但其解释对象应是“新版平台中的已复现能力”，而非旧版原型。
