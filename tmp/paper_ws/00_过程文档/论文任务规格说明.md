# 论文任务规格说明

## 目标

- 项目名称：`采区智能规划设计项目`
- 论文标题：`采区智能规划设计一体化方法与系统`
- 目标期刊：`煤炭科学技术`
- 稿件定位：`方法-系统融合型中文技术论文`
- 本轮目标：在保留现有样例验证边界的前提下，将原有“系统说明稿”重构为“问题驱动、方法清晰、图文对应、可继续投稿深化”的重构稿。

## 研究对象边界

- 当前主系统：`frontend + backend_python` 所对应的新版采区智能规划设计平台。
- 历史原型：`backend/` 与 `server/` 中的 Node/Express 设计原型，仅作为演进背景，不承担正文主叙事。
- 证据来源：仓库代码、已有样例运行结果、接口结果 JSON、已有 PNG/SVG 图件、已整理的中文与英文参考文献池。
- 本轮不纳入范围：
  - 真实矿井多案例对比实验。
  - ODI 参数在不同矿区条件下的系统标定。
  - GNN 模块的泛化性能结论。
  - 作者、基金、通信作者等最终投稿元信息定稿。

## 写作主线

1. 从采区规划设计流程割裂这一工程问题切入，说明现有研究在多场景风险统一表征、规划与接续联动、经济闭环方面的不足。
2. 以一体化方法链为核心组织正文，而不是以代码目录或产品版本为主角。
3. 以“数据标准化与参数场构建 - ODI 风险表征 - 四模式规划 - 三阶段接续 - 经济评价”作为方法主线。
4. 以样例级验证证明链路贯通性和平台级工程可行性，并严格说明证据边界。

## 关键交付

- 重构后的论文工作区。
- 主文图体系与配图清单。
- 论文 Markdown 稿。
- 论文 DOCX 稿。
- 投稿格式对照表、引文映射表、参考文献候选池。
- 新版系统边界说明、系统对象巡检记录、证据矩阵。
